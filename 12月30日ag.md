## 1. 项目概述

### 1.1 核心创新点

**GRA-CNN** (Gray Relational Analysis for CNN Pruning) 提出了一种基于**灰色关联分析**的结构化通道剪枝方法。

**关键创新**:
1. **语义对齐剪枝**: 不同于传统基于权重幅度(L1-Norm)或几何关系(FPGM)的方法，GRA-CNN测量通道激活序列与分类logits之间的**语义关联度**
2. **灰色关联系数**: 使用GRA计算每个通道对最终决策的贡献度，保留与分类边界高度相关的通道
3. **尺度不变性**: GRA对激活值的幅度和分布变化具有鲁棒性

### 1.2 核心公式

```
GRA Score = mean_over_batch(γ)

γ = (Δ_min + ρ·Δ_max) / (Δ_i + ρ·Δ_max)

其中:
- Δ_i = |activation_i - logit_reference|  (归一化后的差异)
- ρ = 0.5 (区分系数，控制对差异的敏感度)
- Δ_min, Δ_max 是所有通道差异的最小/最大值
```

### 1.3 与其他方法对比

| 方法 | 剪枝依据 | 优点 | 缺点 |
|------|----------|------|------|
| L1-Norm | 权重幅度 | 简单快速 | 忽略语义重要性 |
| FPGM | 几何中心距离 | 考虑冗余性 | 仍是结构性标准 |
| HRank | 特征图秩 | 考虑信息量 | 计算复杂 |
| **GRA-CNN** | 语义关联度 | **直接对齐决策边界** | 需要前向传播 |

## 3. 核心文件详解

### 3.1 GRA评分器 (`pruning/gra_score.py`)

```python
# 核心类: GrayRelationalChannelScorer
# 关键方法: compute_score(feature_maps, logits, targets)
# 
# 工作流程:
# 1. 对feature_maps做Global Average Pooling得到[B, C]
# 2. 提取ground truth类别对应的logit作为参考序列
# 3. 对激活序列和参考序列做min-max归一化
# 4. 计算差异序列Δ = |activation - reference|
# 5. 应用GRA公式计算关联系数γ
# 6. 对batch取平均得到每个通道的重要性分数
```

### 3.2 剪枝执行器 (`pruning/prune_model.py`)

负责:
- 根据评分器输出的分数对通道排序
- 按指定ratio移除低分通道
- 重建网络结构并迁移权重

### 3.3 主实验脚本 (`run_all.py`)

```python
# 用法: python run_all.py --arch resnet20 --dataset cifar10 
#                         --method gra --ratio 0.5 --rho 0.5

# 完整流程:
# 1. 加载预训练baseline模型
# 2. 使用指定method计算通道重要性
# 3. 按ratio剪枝
# 4. Fine-tune恢复精度
# 5. 保存结果到CSV和.pth
```
